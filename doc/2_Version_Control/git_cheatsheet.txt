git commit
 - saves changes to local repo
 - -m 'some bs idk' arg lets you add a message

git branch
 - edits branches
 - adding a branch name creates a branch with that name

git checkout 
 - switches current branch
 - with a branch name, it moves to that branch
 - or with -b branch name, it creates a branch with that name and moves to it

git pull
 - fetches changes from remote & merges with current branch
 - --rebase rebases rather than merging 

git push
 - uploads commit to remote repo
 - -u sets a default branch ? idk

git merge
 - combines arg branch history into current branch

git rebase
 - re-applies commits to arg branch rewrites history (unlike merge which combines... ig?)

git fetch
 - downloads everything from remote repo but doesn't merge anything

git add
 - adds file args for next commit.. something something staging area

git status
 - shows what's been staged/modified/untracked in the staging area (for next commit)

git log
 - shows commit history of repo
